# Contributing to hummbl-toolkit

Thank you for your interest in contributing to hummbl-toolkit.

## How to contribute

1. Fork the repository
2. Create a feature branch: `git checkout -b feat/your-description`
3. Commit your changes using Conventional Commits format
4. Push to your fork and open a Pull Request

## Commit conventions

- Use Conventional Commits: `feat:`, `fix:`, `docs:`, `refactor:`, `test:`, `chore:`
- Keep commits focused and atomic
- Reference issues in commit messages when applicable

## Code standards

- Follow existing code style in the repository
- Add tests for new functionality
- Ensure all tests pass before submitting a PR

## Review process

All PRs require review by a code owner. CI must be green before merge.
