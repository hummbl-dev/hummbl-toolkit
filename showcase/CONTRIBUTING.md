# Contributing Guidelines

Thank you for your interest in contributing to HUMMBL!

## How to Contribute

1. Fork the repository and create a feature branch: `type/agent/short-desc`
2. Make focused changes — one concern per commit
3. Ensure tests pass and the codebase follows existing conventions
4. Open a pull request with a clear description

## Conventions

- Conventional Commits (`feat:`, `fix:`, `docs:`, `chore:`, `test:`)
- Imperative commit subjects, ≤ 70 characters
- Small diffs (< 300 lines changed per PR when possible)

## Code of Conduct

All contributors are expected to adhere to our [Code of Conduct](CODE_OF_CONDUCT.md).
